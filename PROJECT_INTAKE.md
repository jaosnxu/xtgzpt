# xtgzpt 协同工作平台

这个仓库用于接收和审查协同工作平台的 PRD / 开发任务。

工作方式：

1. 先把完整开发内容提交到 GitHub Issue。
2. Loop 先做需求审查，不直接开发。
3. 审查输出：
   - 已明确内容
   - 缺失内容
   - 风险
   - 第一阶段可实现范围
   - 需要确认的问题
   - 建议的项目结构和 backlog
4. 确认后再进入开发分支。

提交入口：

- GitHub Issues -> New issue -> `PRD / 开发任务审查`

原则：

- 本仓库只放协同工作平台项目内容。
- 不和 Loop 系统仓库混用。
- 不和其他业务项目混用。
- 未审查前不进入开发。

# TEST ACCEPTANCE STANDARD

本文件定义 `xtgzpt` 的测试和验收标准。

## 1. 总原则

第一版验收不是看页面是否存在，而是看闭环是否真实成立。

每个阶段必须证明：

- 有明确范围。
- 有不做范围。
- 有权限验证。
- 有审计验证。
- 有状态验证。
- 有 API 或浏览器验证。
- 有失败记录。
- 有 reviewer gate。

## 2. P0 阻断项

出现任一情况，不允许合并或进入生产：

- 普通用户看到系统设置。
- 普通用户看到冻结外一级菜单。
- 新增财务、采购、库存、销售、资产、报表、ERP 一级模块。
- AI 自动审批。
- AI 自动发布知识。
- AI 自动创建正式任务。
- AI 自动签署合同。
- AI 自动付款。
- AI 自动确认执行完成。
- 后端接口越权返回数据。
- 文件 URL 猜测可下载。
- 合同没有人工确认就进入审批。
- 审批不是当前节点处理人也能处理。
- 关键动作没有审计。
- 生产密钥进入仓库。
- 生产数据只存在内存中。

## 3. P1 阻断项

P1 必须在当前阶段修复，或明确进入下一阶段 blocker：

- 页面主流程无法完成。
- 角色权限和文档不一致。
- 审批权限仍无法区分节点处理人。
- AI 输出没有来源引用。
- 审计记录缺少对象或操作者。
- 响应式宽度核心任务不可用。
- 错误信息泄露敏感对象名。
- 数据库迁移不可重复执行。

## 4. 每阶段本地 Gate

每个阶段至少运行：

```bash
npm run lint
npm run typecheck
npm run test
npm run build
npm run smoke:api
npm audit --audit-level=low
git diff --check
```

若阶段涉及前端主流程，还必须有浏览器验证。

若阶段涉及数据库，还必须有 repository 或迁移测试。

若阶段涉及 AI，还必须有无 Key 降级测试和 mock provider 测试。

## 5. GitHub Gate

GitHub required checks 必须包括：

- `lint`
- `typecheck`
- `test`
- `build-smoke`
- `audit`

合并前必须满足：

- required checks 全绿
- PR 无未解决 review thread
- 至少 1 个有写权限 reviewer approve
- dev-log 已写
- 未完成范围已记录

## 6. 权限验收

必须覆盖：

- 菜单权限
- 数据权限
- 操作权限
- 审批权限
- 文件权限
- AI 权限

验收用例：

- 普通员工不能访问系统设置。
- 系统管理员可以配置系统，但不默认拥有全部业务数据。
- 非项目成员不能读取项目详情。
- 非聊天成员不能读取聊天内容。
- 非当前审批人不能做审批决策。
- 无文件权限不能预览或下载。
- AI 不能读取无权限来源。

## 7. 审计验收

任一关键动作必须能查询到审计记录。

审计字段至少包含：

- actor
- role
- action
- object type
- object id
- organization id
- result
- reason
- request id
- occurred at
- AI framework version when AI involved

## 8. 业务闭环验收

### 项目

- 创建项目
- 添加成员
- 成员可见
- 非成员不可见
- 状态流转
- 归档后默认列表不展示
- 审计完整

### 任务

- 创建任务
- 负责人推进
- 提交完成
- 确认人确认
- 取消必须写原因
- 状态流转审计完整

### 聊天

- 创建会话
- 成员发消息
- 非成员不可见
- AI 整理有来源
- AI 草稿不能直接变正式对象

### 知识

- 权限过滤检索
- AI 草稿人工确认
- 知识管理员审核
- 来源证据可追踪

### 合同

- 上传或粘贴
- AI 审查
- 风险清单
- 原文标红
- A/B/C 方案
- 人工确认
- 二次审查
- 审批
- 执行跟踪
- 审计完整

### 审批

- 发起
- 当前节点处理人
- 同意
- 驳回
- 退回
- 转交
- 加签
- AI 建议只读
- 结果写回来源对象
- 审计完整

## 9. 响应式验收

必须覆盖：

- 1440px
- 1280px
- 960px

每个宽度必须能完成：

- 查看首页待办
- 查看我的工作台
- 查看项目详情
- 查看任务详情
- 使用聊天和 AI 整理
- 知识检索或问答
- 合同风险确认
- 审批处理
- 系统设置管理

## 10. 生产上线验收

生产上线前必须提供：

- release checklist
- 数据库 migration 记录
- 生产 secrets 配置证明
- 生产 smoke test 结果
- 权限矩阵测试结果
- 审计矩阵测试结果
- AI 边界测试结果
- 文件权限测试结果
- 合同闭环测试结果
- 审批闭环测试结果
- 备份恢复说明
- 回滚方案

## 验收标准

- 关键动作可以回答：谁、什么时候、对象、动作、原因、结果、request id
- 登录成功和登录失败必须写入审计
- 无权限访问必须写入审计
- 管理员 / 超级管理员可按权限查询审计日志
- 对象和用户维度可以查询审计记录
- 猜测的敏感对象名不能进入审计日志
- 未开发业务路由继续返回 `501 not_implemented`
- 本地 `npm run ci` 通过

## 发现与处理

### DEV-004-P2-001 测试用例错误假设系统管理员可看全部审计

现象：

- 初始测试使用 `admin` 查询 owner 所属组织的项目审计
- API 正确按组织数据范围过滤，测试失败

处理：

- 对象审计和用户审计的跨组织验证改用 `super`
- 保留 admin 作用域限制验证

### DEV-004-P3-001 手工验证脚本读取旧登录响应结构

现象：

- 手工脚本读取 `session.token`
- 实际登录响应为顶层 `token`

处理：

- 修正验证脚本
- 记录为验证脚本问题，不是产品接口问题

### DEV-004-P3-002 手工验证脚本读取旧 501 字段

现象：

- 手工脚本断言 `nextStage`
- 实际响应为 `{ "stage": "DEV-005" }`

处理：

- 修正验证脚本
- 记录为验证脚本问题，不是产品接口问题

### DEV-004-P3-003 `localhost:3001` 被无关 Next 服务占用

现象：

- `localhost:3001` 返回旧茶饮系统页面
- 当前仓库 Vite 服务监听 `127.0.0.1:3001`

处理：

- 本阶段浏览器验证统一使用 `http://127.0.0.1:3001`
- 未停止无关进程，避免影响其他项目

### DEV-004-P3-004 Browser 插件截图超时

现象：

- `tab.screenshot({ fullPage: true })` 和视口截图均超时

处理：

- 将 in-app browser 显示后使用系统 `screencapture` 保存截图
- DOM 验证仍由 Browser 插件完成

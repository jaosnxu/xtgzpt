## 目标

产品化既有 Phase 1 前端交互层，使工作入口、详情页和关系证据能直接跳到相关对象，同时保持冻结一级菜单、AI 人工确认边界、权限边界和非生产切流约束。

## 范围

- Dashboard / Workbench 行和通知支持跳转到相关项目、任务、合同、审批和 AI 草稿。
- 项目详情补强到任务、项目文件、关联聊天和项目知识上下文的入口。
- 任务行支持回到来源项目。
- 聊天详情展示关联项目 / 任务 / 合同，并支持跳关联对象、带项目上下文检索和打开 AI 草稿确认结果。
- 知识查询新增项目选择器，保持选择优先、输入最小。
- 合同审批 handoff 可打开审批实例。
- 审批详情可返回来源合同。
- 系统设置可跳到权限治理、AI 治理和页面状态治理面板。
- 增加操作反馈、disabled / loading / error / empty / archived / no-permission 状态细节。
- 增加本地 LAN 验收脚本和 README 指南：API/Web 可用 `0.0.0.0` 监听，另一台同 LAN 电脑通过 Mac LAN IP 打开 Web。

## 不做

- 不新增一级菜单。
- 不扩展 ERP、财务、采购、库存、销售、资产、报表或移动端。
- 不连接生产系统、不提交 secret、不部署、不执行生产切流。

## LAN 本地验收

Mac 上两个终端分别运行：

```bash
npm run dev:api:lan
npm run dev:web:lan
```

查询 Mac LAN IP：

```bash
ipconfig getifaddr en0
```

同一局域网另一台电脑打开：

```text
http://<Mac-LAN-IP>:3001
```

该方式只用于本地验收，不是生产部署或生产切流。

## 范围

- 在 `apps/api/src/runtime-store.ts` 增加 runtime store mode selection。
- 保持测试默认 in-memory store。
- 保持 file runtime store，并继续支持 `XTGZPT_RUNTIME_DATA_FILE`。
- 增加 `XTGZPT_RUNTIME_STORE_MODE=postgres` 的 PostgreSQL runtime boundary。
- 增加 PostgreSQL runtime config validation：
  - `XTGZPT_RUNTIME_DATABASE_URL` 或 `DATABASE_URL`
  - PostgreSQL URL scheme
  - 拒绝 placeholder
  - schema/table safe identifier
  - runtime document id
- 新增 PostgreSQL cutover boundary migration：`0011_runtime_store_cutover_boundary.sql`。
- 增加聚焦测试覆盖 mode resolution、file fallback、PostgreSQL config validation、missing-env failure 和 no-live-write boundary。
- 更新 `.env.example`、runbook、项目标准、backlog、runtime memory 和 run summary。

## 执行结果

- `memory` / `file` / `postgres` 三种 runtime store mode 已进入类型和解析逻辑。
- `NODE_ENV=test` 默认解析为 `memory`，避免测试污染本地文件或数据库。
- 非测试默认解析为 `file`，未设置 `XTGZPT_RUNTIME_DATA_FILE` 时继续使用应用目录下默认 runtime JSON 文件。
- `XTGZPT_RUNTIME_STORE_MODE=postgres` 会在启动边界校验数据库配置；缺少 URL、placeholder、非 PostgreSQL URL 或不安全 identifier 会安全失败。
- PostgreSQL adapter boundary 当前不执行 live database writes；`save()` 明确失败，避免误把 DEV-020 当作真实生产切流。
- 新增 `runtime_data_documents` migration，作为当前 `RuntimeData` JSON shape 的 cutover document boundary。

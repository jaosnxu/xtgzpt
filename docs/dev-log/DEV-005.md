## 验收标准

- 项目负责人可以创建项目
- 普通成员不能创建项目
- 项目负责人可以添加项目成员
- 项目成员可以看到被授权项目
- 未授权用户猜测项目详情返回 404 且不泄露项目名称
- 项目负责人可以创建任务
- 任务负责人可以将任务推进到 `in_progress` 和 `submitted`
- `submitted` 状态必须由确认人或项目负责人确认完成
- 取消任务必须填写原因
- 完成项目后可以归档，归档项目不再出现在列表
- 项目 / 任务关键动作写入对象审计
- `/modules/projects` 和 `/modules/tasks` 返回 `available`
- 本地 `npm run ci` 通过

## 发现与处理

### DEV-005-P2-001 模块状态接口未同步 DEV-005 实现状态

现象：

- 项目 / 任务接口和前端已经接入真实数据
- `/modules/:module` 仍只把 `dashboard` 和 `settings` 标记为 `available`

处理：

- 将 `projects` 和 `tasks` 纳入可用模块列表
- 增加测试覆盖 `/modules/projects` 和 `/modules/tasks`

### DEV-005-P3-001 Playwright 文本选择器命中重复项目名

现象：

- 第一次浏览器验证使用 `getByText("浏览器验证项目")`
- 页面列表和详情标题同时包含该文本，Playwright strict mode 报错

处理：

- 改用 `getByRole("heading", { name })` 和行级 `.task-row` 选择器
- 重新运行浏览器验证通过
- 记录为验证脚本选择器问题，不是产品功能问题

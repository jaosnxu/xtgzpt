## Reviewer 处理记录

独立 reviewer 发现：

- P0：API session token 可伪造
- P1：前端直接选择 seed 用户，绕过 API 登录
- P2：`own_records` 和 `assigned_organizations` 语义混同
- P2：数据库角色权限字段无法表达组织管理 / 角色管理细分权限

处理结果：

- API session token 改为服务端签发并只接受 `sessions` 中存在的 token
- 补 forged token 测试：`Bearer dev-session:user-admin` 必须返回 401
- 前端登录改为调用 API，不再导入 `seedUsers` 或明文密码用户数据
- Vite 本地代理 `/api` 转发到 `127.0.0.1:3002`
- `own_records` 不再拥有组织级业务数据
- 补 member own-record 测试：业务组织列表为空
- 数据库 `roles` 表补 `can_manage_organizations` 与 `can_manage_roles`

第二轮 reviewer 发现：

- P1：登录页仍列出高权限账号，并预填可用密码。

处理结果：

- 登录页改成用户名 / 密码输入表单
- 移除高权限账号快捷按钮
- 移除默认密码
- `seedUsers` 不再包含 password 字段
- dev credential 仅保留在 API 内部

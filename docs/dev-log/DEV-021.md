# DEV-021 Real PostgreSQL Runtime Adapter

时间：2026-06-20T16:06:03Z

## 目标

将 DEV-020 的 PostgreSQL runtime boundary 升级为真实 driver-backed runtime adapter，使 `XTGZPT_RUNTIME_STORE_MODE=postgres` 能通过 PostgreSQL-compatible client 读取、初始化和写入当前 `RuntimeData` document。

## 范围

- 新增 `pg` driver 依赖。
- 将 `RuntimeStore` 生命周期扩展为 `ready` / `save` / `close`。
- PostgreSQL mode 通过连接池读取 `runtime_data_documents`。
- 缺失 document 时自动初始化空 RuntimeData document。
- 写入时使用 checksum 条件更新，避免并发覆盖。
- 读写失败或 checksum 冲突时显式失败，不静默回退。
- 保留 `memory` 和 `file` mode；测试默认仍为 `memory`。
- 增加 mocked PostgreSQL adapter 测试。
- 更新生产 readiness runbook、项目状态文档和 runtime memory。

## 非目标

- 不执行真实生产切流。
- 不提交真实生产 secrets、数据库 URL、token 或账号密码。
- 不新增业务模块、一级菜单、UI 改版或权限扩张。
- 不扩展财务、ERP、采购、库存、销售或经营报表系统。

## 实现摘要

- `apps/api/src/runtime-store.ts`
  - 新增 `PostgresRuntimeClient` / query result 接口，支持真实 `pg.Pool` 或测试替身 client。
  - `createPostgresRuntimeStore` 初始化缺失 document，读取 RuntimeData，并在 save 时 checksum 条件更新。
  - `createPostgresRuntimeStoreForTest` 供测试使用，不需要真实数据库。
- `apps/api/src/index.ts`
  - API 启动时等待 `store.ready`。
  - 关键写入路径改为等待 async `store.save()`。
  - API close 时关闭 runtime store。
- `apps/api/src/runtime-store.test.ts`
  - 覆盖 PostgreSQL 初始化、保存、加载、checksum 冲突和读写失败。
- `apps/api/package.json` / `package-lock.json`
  - 新增 `pg` dependency。

## 发现问题

### AUDIT-002-P1-001 未开发模块存在假入口

现象：

- 主导航展示项目、任务、聊天、知识库、合同、审批等入口
- `新建审批` 按钮看起来可执行
- 但真实项目/任务/聊天/知识库/合同/审批流程尚未进入代码开发

风险：

- 用户会误以为审批闭环已经完成
- 阶段验收容易把“菜单存在”误判为“流程完成”

处理：

- 导航改成真实切换当前模块视图
- 未开发模块显示对应 DEV 阶段和不可执行状态
- `新建审批` 改为 `查看审批阶段`
- 审批模块明确标注 DEV-010 未开发
- 审批发起、当前节点审批人、同意/驳回/退回、AI 建议均标注未开发

### AUDIT-002-P1-002 首页仍展示模拟业务事实

来源：

- 独立 reviewer gate

现象：

- 首页阶段处理队列仍展示“合同待二次审查”“任务完成确认”等模拟事项
- 最近审计仍展示“项目 A 新增成员，已写审计”“合同 OCR 完成”等模拟事实
- 但真实审计写入、项目、任务、合同和审批流程尚未开发

风险：

- 用户和 reviewer 会误以为真实业务流、OCR、任务状态和审计写入已经可用
- 与“真实审计写入将在 DEV-004 接入”的阶段边界冲突

处理：

- 首页处理队列改为 `阶段门检查`
- 只展示认证、角色菜单、组织数据范围等已验证范围
- 审批真实流程明确显示 `DEV-010 待开发`
- 最近审计改为 `审计边界`
- 删除所有模拟业务审计事件

## 4. P1 问题

### P1-001 表格是伪表格，使用空格模拟列

影响页面：

- `P001 首页 / 1440`
- `P002 我的工作台 / 1440`
- `P003 项目 / 列表与详情`
- `P004 任务 / 状态流转`
- `P008 审批详情 / 人工决策`
- `P009 系统设置 / 权限`
- `P007 合同 / 执行跟踪`
- `P008 审批 / 列表与异常动作`
- `P001 首页 / 1280 Standard`
- `角色可见性验收`

问题：

当前表格用一个文本节点和多个空格模拟列，导致：

- 不像正式后台
- 俄文更长时必然错位
- 1280 / 960 下容易溢出
- 开发时无法映射为真实表格组件

处理标准：

- 所有表格必须改为真实列结构。
- 每列必须有固定或弹性宽度。
- 状态列使用 badge，不使用空格。
- 行内操作用按钮或菜单，不拼在文本里。

### P1-002 部分文字超出父容器

影响页面：

- `P001 首页 / 1440`
- `P002 我的工作台 / 1440`
- `P003 项目 / 列表与详情`
- `P008 审批详情 / 人工决策`
- `P009 系统设置 / 权限`
- `P007 合同 / 执行跟踪`

问题：

部分 row 文案实际宽度超过父级 table 容器。

处理标准：

- 列内文字必须可换行或省略。
- 列容器不能依赖字符串空格。
- 长标题必须进入详情页，不在列表中强行铺满。

### P1-003 系统设置页面没有明确管理员导航

影响页面：

- `P009 系统设置 / 权限`
- `系统设置字段级配置`

问题：

页面标题是系统设置，但左侧普通业务菜单不包含系统设置入口，也没有明确的管理员设置二级导航。

处理标准：

- 普通用户仍然只能看到 8 个业务菜单。
- 管理员视图必须有独立的设置模块导航。
- 设置模块应分为：组织、角色、权限、流程、AI 框架、审计。

### P1-004 Phase Scope 长文本拥挤

影响页面：

- `Phase Scope`

问题：

阶段范围说明过长，当前卡片宽度不足，信息密度偏高。

处理标准：

- 改成分组 checklist。
- 避免大段说明塞进一个卡片。

## 5. P2 问题

### P2-001 角色 badge 宽度过窄

影响：

多个页面顶部 badge 出现宽度风险，例如：

- 当前角色：普通员工
- 当前角色：项目负责人
- 当前角色：知识管理员
- 角色：合同发起人
- 角色：法务审批人

处理标准：

- 角色 badge 宽度至少 150px。
- 后续俄文界面需预留 220px 以上。

### P2-002 长说明文字需要拆分

影响：

- 知识库规则说明
- AI 回答证据说明
- 合同修改对比
- 聊天附件说明

处理标准：

- 长说明分区展示。
- 关键状态、来源、证据、动作分开。

### P2-003 Figma 点击连线仍不完整

已完成第一批连线，但还需要补：

- 项目 -> 任务
- 聊天 -> 任务草稿
- 知识问答 -> 知识审核
- 设置模块之间切换
- 无权限返回首页

## 6. 立即修复策略

先修 P1：

1. 新增 QA Fixed 页面，建立正式表格/列表组件样式。
2. 用真实列结构重做首页、审批列表、设置页。
3. 明确管理员设置二级导航。
4. 加宽角色 badge。
5. 把长文本改为分组 checklist。

随后再回到原页面替换或标记旧版废弃。

## 8. 当前结论

问题不是业务方向错，而是 UI 交付质量和结构化组件表达不够正式。

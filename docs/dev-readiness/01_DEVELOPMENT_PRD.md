# DEVELOPMENT PRD

## 1. 产品定位

xtgzpt 是企业内部协同工作平台，用于把项目、任务、沟通、知识、合同、审批放在同一套工作系统里。

核心定位：

- 协同工作平台
- AI 辅助工作平台
- 权限和审计严格的内部系统

非定位：

- ERP
- 财务系统
- OA 大杂烩
- 文档网盘
- AI 自动办公机器人

## 2. 用户角色

第一版至少支持以下角色：

- 普通员工
- 项目负责人
- 部门负责人
- 管理层
- 合同发起人
- 法务审批人
- 财务审批人
- 审批人
- 知识管理员
- 系统管理员
- 超级管理员

AI 不是角色，只是工具能力。

## 3. 核心模块

### P001 首页

目的：让用户看到今天需要处理的工作。

必须包含：

- 我的待办
- 待审批
- 待确认合同风险
- AI 待确认项
- 重要任务
- 重要项目更新
- 快捷入口
- 系统内通知

禁止做成经营数据大屏。

### P002 我的工作台

目的：聚合“和我有关”的工作。

必须包含：

- 我负责的任务
- 我参与的项目
- 我发起的审批
- 我待处理的审批
- 我待处理的合同
- 我需要确认的 AI 结果
- 最近处理记录

### P003 项目

目的：项目协同，不是 ERP 项目后台。

必须包含：

- 项目列表
- 项目详情
- 项目状态
- 负责人
- 成员
- 关联任务
- 评论
- 附件
- AI 风险建议
- 关联审批
- 操作记录

### P004 任务

目的：任务分派、跟进、交付确认。

必须包含：

- 任务列表
- 任务详情
- 负责人
- 截止日期
- 状态流转
- 评论
- @ 提醒
- 附件
- AI 任务草稿
- 关联项目
- 关联审批
- 操作记录

AI 可以生成草稿，但不能自动创建正式任务。

### P005 聊天

目的：内部工作沟通。

必须包含：

- 一对一聊天
- 群聊
- 消息记录
- @ 提醒
- 附件
- 关联项目/任务/合同
- AI 整理
- AI 生成任务草稿
- AI 生成知识草稿

AI 不能删除原始聊天，不能自动发布知识，不能自动创建正式任务。

### P006 知识库

目的：AI 问答式知识入口。

必须包含：

- 知识问答
- 权限过滤后的来源检索
- 答案
- 证据来源
- 来源引用
- 人工确认发布
- 知识管理员审核

禁止做成传统文档库首页。

### P007 合同

目的：合同风险审查和审批执行跟踪。

必须包含流程：

`上传/粘贴 -> AI审查 -> 风险清单 -> 原文标红 -> A/B/C方案 -> 人工确认 -> 修改 -> 二次审查 -> 审批 -> 执行跟踪 -> 审计留痕`

必须包含：

- 合同上传
- 合同粘贴
- AI 结构化审查
- 风险清单
- 原文标红
- A/B/C 修改方案
- 人工确认
- 二次审查
- 审批关联
- 执行跟踪
- 版本记录
- 审计记录

### P008 审批

目的：人类决策工作台。

必须包含：

- 待我审批
- 我发起的审批
- 审批详情
- 来源对象
- 当前节点
- 当前审批人
- AI 建议
- 审批历史
- 审计记录
- 同意
- 驳回
- 退回修改
- 转交
- 加签
- 查看来源

AI 不能成为审批人，不能自动审批。

### P009 系统设置

仅管理员可见。

必须包含：

- 组织管理
- 角色管理
- 权限管理
- 菜单权限
- 数据权限
- 操作权限
- 审批权限
- 文件权限
- AI 权限
- 流程配置
- AI 框架配置
- AI 知识库配置
- AI 信息中心配置
- AI 关联中心配置
- 审计配置

## 4. 核心业务闭环

### 4.1 任务闭环

`创建草稿 -> 分配负责人 -> 执行中 -> 提交完成 -> 人工确认 -> 完成 -> 审计记录`

### 4.2 项目闭环

`创建项目 -> 加入成员 -> 拆解任务 -> 协同沟通 -> 跟踪风险 -> 处理审批 -> 项目完成 -> 审计记录`

### 4.3 聊天到任务闭环

`聊天沟通 -> AI 整理 -> 生成任务草稿 -> 人工确认 -> 正式任务 -> 执行跟踪`

### 4.4 知识问答闭环

`用户提问 -> 权限过滤来源 -> AI 回答 -> 展示证据 -> 人工确认 -> 可沉淀知识`

### 4.5 合同闭环

`上传/粘贴 -> AI 审查 -> 人工确认 -> 二次审查 -> 审批 -> 执行跟踪 -> 审计`

### 4.6 审批闭环

`发起 -> 节点流转 -> 人工决策 -> 结果写回来源对象 -> 审计记录`

## 5. 全局规则

- 所有业务对象必须有所属组织或可见范围。
- 所有文件必须绑定对象、上传人、权限范围。
- 所有 AI 输出必须绑定框架版本。
- 所有 AI 输出必须标记为建议或草稿。
- 所有正式状态变化必须由人或系统规则触发，不得由 AI 直接触发。
- 所有关键动作必须进入审计。
- 无权限时页面必须显示无权限状态，不得泄露数据。
- 页面必须支持宽屏、标准办公屏、窄浏览器。

# PERMISSION MATRIX

## 1. 权限维度

第一版权限必须覆盖 6 个维度：

1. 菜单权限
2. 数据权限
3. 操作权限
4. 审批权限
5. 文件权限
6. AI 权限

权限必须由后台配置，不允许通过前端隐藏按钮代替真实权限校验。

## 2. 角色定义

| 角色 | 定义 |
| --- | --- |
| 普通员工 | 发起、参与、处理自己相关工作 |
| 项目负责人 | 管理授权项目、任务、成员和风险 |
| 部门负责人 | 查看和处理本部门相关工作 |
| 管理层 | 查看授权范围内整体工作进度和关键风险 |
| 合同发起人 | 发起合同上传、审查和审批 |
| 法务审批人 | 审查合同风险和合同审批节点 |
| 财务审批人 | 参与合同或费用相关审批，不等于财务系统 |
| 审批人 | 根据流程节点进行人工审批 |
| 知识管理员 | 审核和发布知识内容 |
| 系统管理员 | 配置组织、角色、权限、流程和 AI 框架 |
| 超级管理员 | 管理系统级配置和管理员权限 |

## 4. 数据权限规则

| 范围 | 规则 |
| --- | --- |
| 本人 | 用户本人创建、负责、参与、被 @、待处理的数据 |
| 项目 | 用户作为成员、负责人或被授权查看的项目 |
| 部门 | 用户所在部门或被授权部门的数据 |
| 组织 | 用户被授权组织内的数据 |
| 全部 | 仅超级管理员或被明确授权的管理角色 |

系统管理员不默认拥有全部业务数据权限。

## 5. 操作权限

| 操作 | 权限要求 |
| --- | --- |
| 新建项目 | 项目创建权限 |
| 编辑项目 | 项目负责人或授权编辑权限 |
| 关闭项目 | 项目负责人、部门负责人或授权管理层 |
| 新建任务 | 任务创建权限 |
| 分配任务 | 项目负责人、部门负责人或任务管理权限 |
| 完成任务 | 负责人提交，确认人确认 |
| 上传文件 | 对象访问权限 + 文件上传权限 |
| 删除文件 | 上传人未进入正式流程前可删除；正式流程文件只允许归档，不允许物理删除 |
| 发起审批 | 对来源对象有发起审批权限 |
| 审批决策 | 当前节点审批人 |
| 发布知识 | 知识管理员或授权发布人 |
| 修改权限 | 系统管理员或超级管理员 |

## 6. 审批权限

- 审批人必须来自流程配置。
- 审批人必须是人类账号。
- AI 只能给建议，不能成为节点审批人。
- 转交、加签、退回必须记录原因。
- 审批通过后必须写回来源对象。

## 7. 文件权限

文件必须继承所属对象权限，并可增加单独限制。

文件权限必须包含：

- 可见
- 下载
- 预览
- 上传
- 删除/归档
- 引用到 AI

AI 只能读取用户当前有权访问的文件。

## 8. AI 权限

| AI 能力 | 默认要求 |
| --- | --- |
| 聊天整理 | 对聊天有访问权限 |
| 任务草稿 | 对来源聊天/项目有访问权限 |
| 知识问答 | 对来源知识有访问权限 |
| 合同审查 | 对合同有访问权限 |
| 审批建议 | 对审批来源对象有访问权限 |
| 风险提示 | 对来源对象有访问权限 |
| 框架配置 | 系统管理员或超级管理员 |

AI 输出不得突破用户权限范围。

## 9. 无权限处理

无权限时必须：

- 不返回敏感数据
- 页面显示无权限状态
- 记录访问拒绝审计
- 禁止通过 URL 猜测访问

禁止：

- 前端隐藏但接口仍返回数据
- 错误提示泄露对象名称、文件名、合同标题
- AI 引用无权限来源

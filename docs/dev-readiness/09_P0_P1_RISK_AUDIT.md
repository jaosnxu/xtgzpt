# P0 P1 RISK AUDIT

## 1. 风险等级

P0：阻断开发或会导致产品方向错误、安全事故、权限事故、AI 越权。

P1：不阻断启动，但会影响正式交付质量，必须在开发阶段解决。

P2：体验、效率、文案、组件一致性问题。

## 2. P0 风险清单

| 风险 | 当前控制 |
| --- | --- |
| 新增普通用户一级菜单 | 已冻结一级菜单 |
| 系统设置暴露给普通用户 | 权限矩阵已限制 |
| 文件中心/通知中心/AI 中心成为一级菜单 | 已禁止 |
| 知识库变成传统文档库 | 已定义为 AI 问答入口 |
| 合同入口不是上传/粘贴 | 已定义合同入口 |
| 合同缺少 AI 审查 | 已定义状态机 |
| 合同缺少 A/B/C 方案 | 已定义 AI 输出 |
| 合同缺少人工确认 | 已定义阻断规则 |
| 合同缺少二次审查 | 已定义流程 |
| AI 自动审批 | 已禁止 |
| AI 自动发布知识 | 已禁止 |
| AI 自动创建正式任务 | 已禁止 |
| AI 自动改变正式状态 | 已禁止 |
| 无权限内容可见 | 权限矩阵和接口草案已覆盖 |
| 文件无权限 | 文件权限已独立定义 |
| ERP 范围扩张 | 已明确不做 ERP |
| Web 窄视口不能处理核心任务 | 已定义响应式验收 |
| 缺失审计 | 审计矩阵已覆盖 |

结论：按本目录执行时，P0 风险均有明确控制。开发中如偏离，需要立刻阻断。

## 3. P1 风险

| 风险 | 影响 | 控制 |
| --- | --- | --- |
| 权限配置过复杂 | 管理员难以配置 | 设置页面分模块，默认模板 |
| 审批流程配置过弱 | 后续难扩展 | 流程配置保留节点、角色、人员、条件字段 |
| AI 框架版本未固化 | 输出不可追踪 | AI Run 必须记录 framework_id/version |
| 合同标红交互复杂 | 用户看不懂风险 | 合同页必须并列显示原文、风险、方案 |
| 窄浏览器信息拥挤 | 核心操作困难 | 详情页改上下布局，不隐藏关键操作 |
| 系统管理员误认为能看全部数据 | 权限事故 | 管理权限和业务数据权限分离 |
| 知识来源质量不稳定 | AI 回答不可靠 | 显示来源和缺失信息 |
| 审计数据过多难查 | 排查困难 | 按对象、用户、动作、时间过滤 |

## 4. P2 风险

| 风险 | 控制 |
| --- | --- |
| 页面文案不一致 | 建立术语表 |
| 状态颜色不一致 | 建立组件状态规范 |
| 空状态过多 | 用真实任务入口引导 |
| AI 输出过长 | 使用摘要、风险、证据分区 |

## 5. 开发中阻断规则

出现以下情况必须停止继续开发，先回到 issue/PR 处理：

- 需求要求新增被冻结禁止的一级菜单
- 需求要求 AI 自动审批或自动发布
- 需求要求普通用户访问系统设置
- 需求要求跳过合同人工确认
- 需求要求绕过文件权限
- 需求要求做 ERP、财务、采购、库存
- 任一接口返回无权限数据
- 任一关键动作没有审计路径

## 6. 剩余开放问题

这些不是 P0，但进入正式开发前需要产品确认：

- 第一版是否需要组织层级导入模板
- 审批节点是否需要条件分支
- 合同二次审查是否必须使用同一 AI 框架版本
- 知识管理员是否按组织分配
- 聊天记录保留期限
- 文件预览水印是否第一版需要
- 审计日志保留周期

# TEST CASES AND ACCEPTANCE

## 1. 总体验收标准

第一版验收不是看页面是否存在，而是看闭环是否成立。

必须通过：

- 菜单边界测试
- 角色权限测试
- 数据权限测试
- 文件权限测试
- AI 边界测试
- 合同闭环测试
- 审批人类决策测试
- 审计完整性测试
- 页面状态测试
- 响应式测试

## 2. P0 阻断测试

| 用例 | 预期 |
| --- | --- |
| 普通用户访问 `/settings` | 403 或无权限页面 |
| 普通用户看到系统设置菜单 | 不允许 |
| 普通用户看到用户一级菜单 | 不允许 |
| 首页出现文件中心/通知中心/AI 中心一级菜单 | 不允许 |
| AI 直接调用审批通过 | 不允许 |
| AI 自动发布知识 | 不允许 |
| AI 自动创建正式任务 | 不允许 |
| 无权限用户通过 URL 打开合同详情 | 不返回合同内容 |
| 无权限用户通过文件 URL 下载文件 | 不允许 |
| 合同无二次审查直接审批 | 不允许 |
| 合同 AI 审查后无人确认继续流转 | 不允许 |
| 审计记录被删除 | 不允许 |

## 3. 菜单测试

普通员工登录：

- 能看到：首页、我的工作台、项目、任务、聊天、知识库、合同、审批
- 不能看到：系统设置

系统管理员登录：

- 能看到系统设置
- 不默认看到全部业务数据

超级管理员登录：

- 能管理系统级配置
- 业务数据访问仍需符合配置规则

## 4. 项目测试

| 步骤 | 预期 |
| --- | --- |
| 创建项目 | 生成项目审计 |
| 添加成员 | 成员可见项目 |
| 非成员访问项目 | 无权限 |
| 项目下创建任务 | 任务关联项目 |
| 项目完成 | 状态进入 completed |
| 项目归档 | 不在默认列表展示 |

## 5. 任务测试

| 步骤 | 预期 |
| --- | --- |
| AI 从聊天生成任务草稿 | 只是草稿 |
| 人工确认草稿 | 创建正式任务 |
| 负责人提交完成 | 进入 submitted |
| 确认人确认 | 进入 completed |
| 任务取消 | 必须填写原因 |
| 任务状态变化 | 写审计 |

## 6. 聊天测试

| 步骤 | 预期 |
| --- | --- |
| 创建群聊 | 成员可见 |
| 非成员访问 | 无权限 |
| AI 整理聊天 | 有来源引用 |
| AI 生成任务草稿 | 需要人工确认 |
| 撤回消息 | 审计保留 |

## 7. 知识库测试

| 步骤 | 预期 |
| --- | --- |
| 普通用户提问 | 仅检索有权限来源 |
| 无权限来源存在 | 不进入 AI 输入 |
| AI 回答 | 显示证据 |
| AI 生成知识草稿 | 不能自动发布 |
| 知识管理员发布 | 进入 published |

## 8. 合同测试

完整合同闭环：

1. 上传或粘贴合同
2. 发起 AI 审查
3. 展示风险清单
4. 展示原文标红
5. 展示 A/B/C 方案
6. 人工确认风险
7. 修改合同
8. 二次审查
9. 提交审批
10. 人工审批
11. 执行跟踪
12. 查看审计

每一步都必须有状态变化和审计记录。

## 9. 审批测试

| 步骤 | 预期 |
| --- | --- |
| 发起审批 | 进入 submitted/processing |
| 当前审批人查看 | 可以操作 |
| 非当前审批人查看 | 不能操作 |
| AI 生成审批建议 | 只展示建议 |
| 审批人同意 | 进入下一节点或 approved |
| 审批人驳回 | 状态 rejected |
| 转交 | 当前处理人变化 |
| 加签 | 增加节点 |

## 10. 文件测试

| 步骤 | 预期 |
| --- | --- |
| 上传文件 | 绑定来源对象 |
| 预览文件 | 校验文件权限 |
| 下载文件 | 校验文件权限 |
| AI 使用文件 | 校验用户权限 |
| 正式流程文件删除 | 不允许物理删除 |
| 文件归档 | 审计保留 |

## 11. 响应式测试

必须分别在以下宽度验收：

- 1440px
- 1280px
- 960px

每个宽度必须能完成：

- 查看待办
- 查看项目详情
- 查看任务详情
- 查看聊天和 AI 整理
- 知识问答
- 合同风险确认
- 审批处理

## 12. 验收结论

若任一 P0 用例失败，不允许进入生产开发下一阶段。

若 P1 用例失败，可以进入修复 backlog，但必须明确风险和负责人。
